## File Overview

| Property | Value |
|----------|-------|
| **Path** | `/.gitignore` |

| **Lines** | 34 |
| **Bytes** | 313 |
| **Affects** | All `git add` operations |

---

## What It Does

Tells Git which files and directories to ignore when tracking changes.

**Inputs:** None (configuration file)
**Outputs:** Git behavior modification
**Side Effects:** Matching files are excluded from commits

---

## Pattern Breakdown

### Dependencies (Lines 1-3)

```gitignore
# Dependencies
node_modules/
package-lock.json
```

| Pattern | Matches | Reason |
|---------|---------|--------|
| `node_modules/` | npm packages directory | Regenerated by `npm install` |
| `package-lock.json` | Lock file | Contentious; some teams do commit this |

> [!NOTE]
> Excluding `package-lock.json` is a choice. Many projects commit it for reproducible builds.

---

### Environment Files (Lines 5-8)

```gitignore
# Environment files
.env
.env.local
.env.*.local
```

| Pattern | Matches | Reason |
|---------|---------|--------|
| `.env` | Main env file | Contains secrets |
| `.env.local` | Local overrides | Machine-specific |
| `.env.*.local` | Environment-specific local | e.g., `.env.production.local` |

**Security Critical:** Never commit `.env` files with real secrets.

---

### Build Output (Lines 10-13)

```gitignore
# Build output
dist/
build/
out/
```

| Pattern | Matches | Reason |
|---------|---------|--------|
| `dist/` | Distribution build | Regenerated by build |
| `build/` | Build output | Regenerated |
| `out/` | Output directory | Regenerated |

---

### Logs (Lines 15-18)

```gitignore
# Logs
logs/
*.log
npm-debug.log*
```

| Pattern | Matches | Reason |
|---------|---------|--------|
| `logs/` | Log directory | Runtime artifacts |
| `*.log` | Any log file | Runtime artifacts |
| `npm-debug.log*` | npm debug logs | Debugging artifacts |

---

### OS Files (Lines 20-23)

```gitignore
# OS files
.DS_Store
Thumbs.db

```

| Pattern | Matches | Reason |
|---------|---------|--------|
| `.DS_Store` | macOS folder metadata | OS-specific |
| `Thumbs.db` | Windows thumbnails | OS-specific |
| `Desktop.ini` | Windows folder settings | OS-specific |

## Error Handling

**Not applicable.** Invalid patterns are silently ignored.

To debug gitignore issues:

```bash
git check-ignore -v <file>
```

---

## How to Modify Safely

1. Test with `git status` before committing
2. Use `git check-ignore` to verify patterns
3. Be careful with wildcards — they may match more than intended

**Adding patterns:**

```gitignore
# Add at end of relevant section
my-new-pattern/
```
